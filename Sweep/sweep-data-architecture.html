Sweep · ארכיטקטורת הנתונים

אין בסיס נתונים רלציוני — כל "טבלה" היא אובייקט JSON ב-localStorage. מפתח החיבור בכולן הוא fKey = טלפון (או שם כברירת־מחדל).

קשרים: יחיד↔יחיד (1:1) יחיד↔רבים (1:∞) רבים↔רבים (∞:∞) סטטי / חיצוני ◆ מפתח חיבור שדה נסתר מהיוזר

PEOPLE — אנשי הקשר myshare_people / getContacts()

טבלת האב. מקור: אנשי הקשר של המכשיר, עם נפילה ל-localStorage.
phone ◆המפתח הראשי (fKey/statKey). מספר הטלפון הראשי — לפיו הכול מתחבר.
nameשם התצוגה (מפתח חלופי אם אין טלפון).
phones[]כל מספרי הטלפון.
email / emails[]כתובות דוא״ל.
accountחשבון המקור (מכשיר / Google וכו׳).
orgארגון / חברה.
photoURI לתמונת פרופיל.
favמסומן כמועדף (מתמזג מ-FAVSET).
CONTACT_IDנסתרמזהה אנדרואיד — קיים בצד הנייטיב בלבד, לא נשמר בטבלה.
LOOKUP_KEYנסתרמפתח יציב של איש קשר באנדרואיד, לפתיחה/עריכה בספר הטלפונים.

LISTS — רשימות ∞ : ∞

יושבת בתוך cfg (myshare_cfg → lists[]). מקושרת ל-PEOPLE דרך members[] = מערך fKeys. איש קשר יכול להיות בכמה רשימות.
id ◆מזהה הרשימה.
nameשם הרשימה.
members[]מערך fKeys → החיבור לאנשי הקשר.
manual[]חברים שנוספו ידנית.
rules[] / ruleכללי סינון אוטומטי (מילות מפתח).
kitדגלמסמן את רשימת "שמירה על קשר" המיוחדת.
hiddenדגלרשימה מוסתרת מהתצוגה.

ILOG — יומן מגעים 1 : ∞

myshare_ilog · ממופתח לפי fKey. לכל איש קשר — מערך של אירועים.
[fKey] ◆מפתח איש הקשר.
dateחותם זמן של המגע.
kindסוג: share / call / whatsapp / mail…

FIELDS — שדות מותאמים 1 : ∞

myshare_fields · ממופתח לפי fKey. מערך שדות חופשיים לכל איש קשר.
[fKey] ◆מפתח איש הקשר.
lתווית השדה.
vערך השדה.

טבלאות לוויין 1 : 1

כל אחת היא מילון { fKey → ערך } — קשר יחיד־ליחיד עם איש הקשר.
FAVSETmyshare_favset — סימון מועדף.
STATSmyshare_stats — מונה שימוש (לדירוג).
KITmyshare_kit — תדירות שמירה על קשר (מס׳ ימים).
DAYSmyshare_days — מערך 7 בוליאנים (ימי שבוע).
ADDRmyshare_addr — כתובת (לניווט Waze).
BDAYmyshare_bday — יום הולדת (YYYY-MM-DD).
FAVORDERmyshare_favorder — מערך fKeys לסדר המועדפים.

cfg + מצבי־תצוגה הגדרות

myshare_cfg — אובייקט ההגדרות הכללי, ובתוכו יושבות ה-LISTS.
hand / paletteיד דומיננטית, ערכת צבע.
share[] / bank[]פעולות פעילות — מפנות ל-CATALOG לפי id.
lists[]כל הרשימות (ראה כרטיס LISTS).
RECPIN / RECHIDEמצבנעיצה/הסתרה ב"אחרונים" (לפי statKey).
EVHIDEמצבהסתרת אירועים — לפי חתימת אירוע, לא לפי איש קשר.
kitsort / fav_abc / mgdoneUIהעדפות מיון ומצבי אשף.

CATALOG — מאגר פעולות סטטי

רשימת פעולות קבועה בקוד. cfg.bank / cfg.share מפנים אליה דרך id.
id ◆whatsapp · call · sms · mail · copy · telegram…
label / iconשם וסמל להצגה.

יומני מכשיר חיצוני

לא נשמרים באפליקציה — נקראים בזמן אמת מהגשר הנייטיבי. החיבור לאיש קשר לפי 9 הספרות האחרונות של הטלפון (גישור על פורמט 972+/0).
getCallLog{ date, dur, type } — יומן שיחות.
getSmsLog{ date, body, type } — הודעות.
getRecentContacts{ num, date, kind } — מגעים אחרונים.
getNotifLogהתראות (לפי שם).
הרעיון בקצרה: טבלת PEOPLE במרכז, וכל שאר הנתונים תלויים בה דרך מפתח אחד — fKey (טלפון, או שם כגיבוי). טבלאות הלוויין הן 1:1, ILOG ו-FIELDS הן 1:∞, ורק LISTS יוצרת קשר ∞:∞ דרך המערך members[]. יומני השיחות של המכשיר מתחברים בנפרד, לפי 9 הספרות האחרונות של המספר.
