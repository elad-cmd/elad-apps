# Sweep — מפרט עיצוב להעברה (Design Handoff)

סיכום מלא של **כל החלטות העיצוב** לאחר הריוויו. מטרתו: לאפשר למממש (Cowork / Claude Code / מפתח) לתקן את המסכים כך שיהיו אחידים ומקצועיים, בלי לנחש. כל ערך מוגדר במפורש.

---

## 0. איך להשתמש במסמך הזה ב-Cowork

1. פתח סשן חדש ב-Cowork על תיקיית הפרויקט של Sweep.
2. צרף את הקובץ הזה (`sweep-design-handoff.md`) **וגם** את קבצי ה-HTML של הפרוטוטייפים כמקור-אמת ויזואלי (רשימה מלאה בסעיף 6).
3. תן ל-Cowork את הפרומפט הבא:

> "מצורף מפרט עיצוב מלא (`sweep-design-handoff.md`) וקבצי HTML שהם מקור-האמת הוויזואלי. עבור מסך-מסך בקוד של האפליקציה והחל את המפרט: קודם הגדר את ה-Design Tokens כמשתנים מרכזיים במקום ערכים קשיחים, ואז החל אותם על הרכיבים. התחל ממסך אנשי הקשר הראשי, ואז נראו-לאחרונה, שמירה-על-קשר, פרטי-איש-קשר, וימי-הולדת. אחרי כל מסך עצור והצג לי לפני שתמשיך. אל תשנה לוגיקה עסקית — רק שכבת התצוגה."

> **חשוב:** ציין ל-Cowork את שפת המימוש בפועל (SwiftUI / Flutter / React Native / וכו') כדי שימיר את ה-Tokens לפורמט הנכון. ה-Tokens כאן ניטרליים לפלטפורמה.

## 1. עקרונות-על (לא לחרוג מהם)

1. **רגוע, לא צועק.** אפליקציה שמסתכלים בה כל הזמן. אין זוהר, `glow`, `bloom` או ניאון. משטחים שטוחים ומאט.
2. **כהה-רך, לא שחור.** רקע פחם-ירקרק (`#14231F`), טקסט ברוך מופחת-ניגודיות. שחור-כמעט-מוחלט נפסל — מעייף את העין.
3. **טורקיז כנגיעה שקטה.** צבע המותג נשאר, כמילוי מדוד ולא כמקור אור. בידול בא ממרווח ומניגודיות עדינה.
4. **רכיב שורה אחד בכל מקום.** כל מסך שבו מופיעים אנשי קשר משתמש **באותה שורה** בדיוק. משתנה רק "משבצת ההקשר" בקצה (חץ / סטטוס / כפתור).
5. **הסייען הקולי תמיד במרכז.** המיקרופון קבוע במרכז התפריט התחתון בכל מסך. לעולם לא זז.
6. **חתימת 45°.** שלוש נקודות מדורגות באלכסון (קטן שמאל-למטה → גדול ימין-למעלה): הלוגו, האייקון, וכפתור התפריט — אותה צורה בדיוק.

## 3. רכיבי ליבה

### 3.1 Avatar
- עיגול, `border-radius: 50%`. **בלי תמונה:** רקע `avatar-bg`, ראשי-תיבות `avatar-ink` משקל 600. **עם תמונה:** ממלאת (`cover`), מקור = Google של איש הקשר.
- מספר לא מזוהה: רקע `#26332f`, סימן `?` בצבע `text-3`.
- כוכב מועדף: אייקון 17px בפינה, ממולא `accent`.
- ❌ **בלי עיגולים לבנים ענקיים.** זה הפגם המרכזי שהוסר בכל המסכים.

### 3.2 Contact Row (הרכיב המרכזי — בכל מקום)
`[Avatar מימין] [Info: name + meta] [Context Slot משמאל]`
- Info: שם (`font-name`, `text`); מטא (`font-meta`, `text-2`) מתחת. שורה אחת, `ellipsis`.
- מספרי טלפון `dir="ltr"` בתוך תא ה-info.
- מפריד בין שורות: `line`, מתחיל אחרי האווטאר.
- `:active` → רקע `surface`.

### 3.3 Context Slot — מה שמשתנה בין מסכים
| מסך | תוכן המשבצת |
|-----|-------------|
| ראשי / רשימות (משפחה, מדריכים…) | Chevron (`text-3`) או ריק |
| נראו לאחרונה | חץ סוג-שיחה + זמן (בתוך ה-info) |
| שמירה על קשר | צ׳יפ סטטוס (`late` / `ok`) |
| ימי הולדת | כפתור "ליומן" (סעיף 3.11) |
| מיזוג / בחירה | צ׳קבוקס |

> **חוק:** תדירות וסטטוס ("באיחור", "עוד X ימ׳") מופיעים **אך ורק** ב"שמירה על קשר".

### 3.4 Status Chip (שמירה על קשר בלבד)
`padding: 5–6px 11px`, `radius: 9px`, `font 12.5px/600`. "באיחור" = טקסט `late` על `late-bg`; "עוד X ימ׳" = `ok` על `ok-bg`.

### 3.5 Call-type Arrow (נראו לאחרונה בלבד)
- חץ 16px, **בלי טלפון**. הכיוון+הצבע מספרים הכול.
- יוצאת: חץ החוצה (↗), `call-out`. נכנסת: חץ פנימה (↙), `call-in`. לא-נענתה: חץ פנימה, `call-missed`.
- **רק החץ נצבע** — השם נשאר רגיל, בלי תווית מילולית ("נכנסת/יוצאת").
- מונה רצף: `(2)` אחרי השם, `text-3`.

### 3.6 Bottom Dock + חוקי ניווט
- רקע: גרדיאנט מ-`bg` לשקוף, גובה ~78px. כפתור רגיל: `fab-size`, עיגול, `surface`, מסגרת `line-strong`, אייקון `text-2`.
- **מיקרופון (ראשי):** `fab-primary-size`, מילוי `accent-deep`, אייקון בהיר. **תמיד במרכז.**
- **ליבה קבועה בכל מסך:** `Back(ימין) · מיקרופון(מרכז) · תפריט-45°(שמאל)`. הצדדים מתחלפים לפי כיווניות.
- **מסכי אזור אנשי-הקשר** (ראשי, נראו לאחרונה, מועדפים) מרחיבים ל-5: `Back · חייגן · מיקרופון · חיפוש · תפריט-45°` (חייגן=שני מימין, חיפוש=שני משמאל).
- **כפתור תפריט-45°:** שלוש נקודות מדורגות באלכסון (r: 1.7→2.1→2.5), `currentColor`. זהה ללוגו ולאייקון.

### 3.7 Overlay (תפריט / שיטס)
Scrim `rgba(0,0,0,.55–.62)` מקהה את המסך; פאנל `bg-overlay`, פינות עליונות `radius-sheet`, מחליק מלמטה. ❌ בלי פאנלים בהירים-מנטה.

### 3.8 לוגו ואייקון
- **לוגו:** שלוש נקודות `accent` באלכסון 45° **משמאל** למילה "Sweep", מדורגות בגודל+שקיפות (`4px/.4 → 5.5px/.7 → 7.5px/1`), שטוח.
- **אייקון:** ריבוע-מעוגל (radius ~24%), רקע טורקיז עמוק שטוח `#0E8A72`, שלוש נקודות לבנות מדורגות, בלי זוהר. קובץ: `sweep_icon_calm_1024.png`.

### 3.9 Action Button (כפתורי פעולה בכרטיס איש קשר)
- עיגול 44px, רקע `surface-2`, אייקון `accent` (stroke). `:active` → `#31504a`.
- ❌ **לא עיגולים לבנים.** כל אייקוני הפעולה (וואטסאפ/חייג/מייל/וייז/יום-הולדת) בטיפול הזה.
- בשדה ריק: האייקון בצבע `text-3` (מעומעם).

### 3.10 Field Row (מסך פרטי איש קשר)
`[Action button(s) מימין] [Field box] [Copy icon משמאל]`
- **שדה מלא:** מציג את הערך בלבד, **בלי תווית פנימית** ("נייד"/"כתובת"/"תאריך לידה"). האייקון מימין מציין את סוג השדה.
- **שדה ריק:** מציג רמז מעומעם ("הוסף מייל") + אייקון פעולה מעומעם.
- **Copy icon:** עדין, `text-3`, בלי רקע — נוכח אך לא מושך תשומת לב.
- ערך: `dir="ltr"` למספרים/מייל; `dir="rtl"` לכתובת/תאריך עברי.

### 3.11 "ליומן" Button (מסך ימי הולדת)
- פיל: `surface` + מסגרת `line-strong`, אייקון לוח-שנה + הטקסט "ליומן", `font 13px`. פעולה: הוספה/סנכרון היום ליומן.
- בשורת יום-הולדת של היום: הכפתור נצבע `today` (ממולא), טקסט כהה.

## 4. מפרט לפי מסך

### 4.1 מסך אנשי קשר ראשי
- כותרת: לוגו Sweep. טאבים: אחרונים / אנשי-קשר / מועדפים (פעיל = מילוי `accent`).
- צ׳יפי רשימות: הכל / שמירה-על-קשר / משפחה / מדריכים (פעיל = `accent`). ❌ בלי מסגרת אדומה.
- רשימה: שורות, משבצת = chevron. **בלי סטטוס תדירות.**
- דוק 5 כפתורים (סעיף 3.6).

### 4.2 מסך "נראו לאחרונה"
- טאב פעיל = "אחרונים". סגמנט: `הכל` / `שלא נענו`.
- קיבוץ לפי יום: `היום` / `אתמול` / `מוקדם יותר`.
- שורה: אווטאר, שם, חץ סוג-שיחה + זמן. ❌ בלי כפתור חיוג נפרד בשורה — לחיצה על השורה מחייגת.

### 4.3 מסך "שמירה על קשר"
- כותרת + **כפתור מיון (toggle)** למעלה-שמאל: מחליף בין שני מצבי מיון (דחיפות ↔ א״ב/שם).
- שורה: מטא = `תדירות · מגע אחרון`, משבצת = צ׳יפ סטטוס. מיון ברירת מחדל: מאחרים למעלה.
- כפתור תחתון: "+ הוסף אנשים" (`accent-deep`).

### 4.4 מסך "פרטי איש קשר"
- כותרת "פרטי איש קשר". דוק מסך-משנה (3 כפתורים).
- **Hero:** אווטאר גדול (100px, כהה — לא לבן) במרכז, כוכב מועדף בפינה, שם (`font 24/700`), חשבון (מייל, `text-3`) מתחת.
- **שדות:** רכיב Field Row (3.10). שדות מלאים בולטים; ריקים מעומעמים.
- **שדות מותאמים:** "+ הוסף שדה". **ימים פנויים:** 7 עיגולי יום (פעיל = `accent-deep`). שורת "מגע אחרון" עדינה בתחתית.

### 4.5 מסך "ימי הולדת קרובים"
- כותרת "ימי הולדת קרובים". דוק מסך-משנה. בנוי **כמו "שמירה על קשר"**.
- שורה: אווטאר כהה, שם, מטא = ספירה ("בעוד 152 ימים · 03/12"), משבצת = כפתור "ליומן".
- **יום-הולדת היום:** רקע שורה `today-bg`, שם+מטא בצבע `today`, אימוג׳י 🎂 ליד השם, טקסט "היום! · DD/MM", כפתור "ליומן" מודגש.
- מיון: הקרוב ביותר למעלה.

## 5. מצבים, מקרי קצה, נגישות, מושן

### מצבים
- Row `:active` → `surface`. Chip/Tab פעיל → `accent` + `accent-ink`.
- **Empty state** (רשימה ריקה) → אייקון עדין + טקסט `text-2` (להגדרה).
- **Loading** → שלד שורות (skeleton) בגוון `surface`, לא ספינר.

### מקרי קצה
- שם ארוך → שורה אחת, `ellipsis`. בלי תמונה → ראשי-תיבות. מספר לא מזוהה → אווטאר `?`.
- רשימה גדולה (1758 אנשי קשר) → גלילה וירטואלית.

### RTL
- כל מסך `dir="rtl"`. אווטאר בקצה ימין; משבצת הקשר בקצה שמאל. מספרים/לטינית → `dir="ltr"` נקודתי.

### נגישות
- ניגודיות `text` על `bg` ≥ AA (לוודא). `aria-label` עברי לכל כפתור דוק/פעולה. יעד מגע ≥ 44px.

### מושן (עדין בלבד)
| אלמנט | טריגר | אנימציה | משך | Easing |
|-------|-------|---------|-----|--------|
| שיטס/תפריט | פתיחה | החלקה מלמטה + scrim fade | 300ms | `cubic-bezier(.22,1,.36,1)` |
| שורה | לחיצה | שינוי רקע | 150ms | ease |
| טאב/צ׳יפ | מעבר | שינוי צבע | 180ms | ease |
