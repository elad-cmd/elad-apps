# Sweep — המשך עבודה (Handoff, סוף "Sweep 14")

קרא את הקובץ הזה בתחילת כל שיחת Sweep. גרסה נוכחית: **`2026.07.08.0030`** (⚠️ **עדיין לא נפרסה / לא אומתה במכשיר** — צריך `deploy-apps.bat` + "עדכן אפליקציה").

## קבצים ובנייה
- מקור ה-UI: `apps/MyShare.html` (קובץ ענק יחיד). נטען ב-WebView מ-GitHub Pages / מהאסט המובנה `Sweep/android/app/src/main/assets/myshare.html`.
- נייטיב: `apps/Sweep/android/app/src/main/java/com/sweep/ShareActivity.kt` (מחלקת `Bridge` = ממשק JS בשם `AndroidShare`).
- **דיפלוי = דאבל-קליק על `apps/deploy-apps.bat`** (הצג אותו כקרד לחיץ כשמבקשים לפרוס). דוחף את `apps` ל-repo `elad-cmd/elad-apps` → GitHub Actions "Build Sweep APK" בונה APK + מפרסם Release (tag `sweep-latest`) + GitHub Pages.
- **שינויי HTML/CSS** → מגיעים למכשיר דרך "עדכן אפליקציה" (מושך מ-GitHub Pages; עדיין צריך דיפלוי כדי לפרסם). **שינויי Kotlin/נייטיב** → מצריכים בניית APK והתקנה ידנית.
- בכל שינוי: לעדכן `const APP_VERSION=` ב-MyShare.html **וגם** `version.json` יחד.
- חתימת APK = `debug.keystore` מחויב בריפו (יציב → עדכונים לא מוחקים נתונים).
- אפשר לקרוא לוגים של GitHub Actions דרך Claude-in-Chrome (אלעד מחובר) לאבחון כשלי בנייה. תיקון CI שהוחל: `env: ACTIONS_ALLOW_USE_UNSECURE_NODE_VERSION: 'true'` ב-build-sweep.yml.
- **אימות JS: bash רואה עותק מיושן/חתוך של הקובץ ב-Dropbox** (השורות האחרונות נחתכות באמצע!) — אי אפשר להריץ `node --check` אמין על כל הקובץ. לסמוך על כלי ה-host (Read/Grep/Edit) ולאמת כל אזור עריכה מול המקור.
- `fKey`=`statKey`=מספר טלפון גולמי (לא יציב). `healKeys()` (בטעינה) מרפא מפתחות שמורים (members+manual) לפי כל הטלפונים+שם.

## נעשה ב-Sweep 13 (השיחה הזו) — כולל במסך פרטי איש קשר (openContact)
- **אנימציית מסך פרטי → אקורדיון אמיתי:** `scrObjs` פורס עכשיו את ילדי `.ct-card` (במקום כרטיס אחד גדול) לקסקדה; ו-`paintHist` (ההיסטוריה) נדחה ב-`setTimeout(...,80)` כדי שהקריאות הנייטיביות הכבדות (getCallLog/getSmsLog/getNotifLog) לא יחסמו את פריימי הכניסה. פתר את "נכנס/יוצא מאוחר + טקסט-על-טקסט".
- **מולטי-סלקט להיסטוריית מגעים** (`#ctHistory` ב-openContact): לחיצה ארוכה על רשומה → `showSelBar` עם **בחר הכל / הסר** (בלי "העתק"). state מקומי `histSel`/`histSelSet`, מפתח = `evKey(e)` = date|icon|dir. פונקציות: `exitHistSel`, `histRemoveSel`, `updHistBar`.
- **צ'קבוקסים מימין:** בהיסטוריה (flex RTL, ה-`.pk-chk` הוא הילד הראשון). וגם ב**מולטי-סלקט של האחרונים** — הוזז ל-CSS `.row.rec-selrow` (checkbox ל-right:16, `.av` ל-54, `.meta` ל-113). CSS חדש: `.hist-row.sel`.
- **תמונת איש קשר במסך פרטי:** `.ct-av` מקבל עכשיו `position:relative;overflow:hidden` + מציג `person.photo` כמו `avInner`.

## נעשה ב-Sweep 14 — תיקון באג "הרשימה נגללת בלי הטאבים" (pull-to-reach)
הבאג: אחרי חזרה ממסך פרטי למסך הבית, הגלילה-למטה (ה-pull שמוריד את כל המסך) הזיזה רק את הרשימה בלי ה-seg/listbar. סיבת שורש — שלושה כותבי transform מתנגשים: `setPull` מול האנימציות `accIn`/`accOut`. תוקן ב-3 נקודות ב-MyShare.html:
1. **`accIn` (ניקוי ב-1050ms):** המחיקה של transform ל-'' דרסה pull פעיל באמצע מחווה. עכשיו אחרי הניקוי, אם `sc.id==='home'` ו-`pullOffset>0` → `setPull(pullOffset)` מחזיר סנכרון.
2. **`openScreen`:** קודם רק openContact איפס pull; מסכים אחרים (הגדרות/סינון/ארכיון) השאירו `pullOffset>0` בזמן שהאקורדיון דרס את הטאבים → חוסר סנכרון קבוע בחזרה. עכשיו כל פתיחת מסך מ-home מאפסת (`setPull(0)`).
3. **touchmove של ה-pull:** כשמחווה מתחילה, מנקה `transition` גם על seg/listbar (לא רק listEl) כדי שהטאבים יעקבו אחרי האצבע גם בתוך חלון האנימציה של accIn.

## נעשה ב-Sweep 14 (המשך) — שלושה תיקונים נוספים מבדיקות אלעד במכשיר
1. **תפריט המניפה היה מת בכל המסכים הפנימיים** (אלעד גילה במסך סדר פסח): שריד CSS `body:has(.scr) .fab-menu{pointer-events:none!important}` (~שורה 164) מהעיצוב הישן (סרגל נפרד לכל מסך) — הוסר. עכשיו הכפתור הצף פעיל גם במסכים.
2. **קריסה "innerHTML of null" ב-openNormalizePreview** (בלחיצה מהירה על "ניקוי מספרים" מתוך סדר פסח): הגארד נגד דאבל-טאפ ב-`openScreen` החזיר את המסך הקיים (הלא-נכון) בתוך חלון navLock/380ms. תוקן: המסך נושא `sc._html`, והגארד מחזיר מסך קיים רק אם ה-HTML זהה (דאבל-טאפ אמיתי); אחרת פותח רגיל. מגן על כל המסכים.
3. **מיזוג כפילויות — חיפוש למטה:** המבנה שונה ל-flex טור עם מחלקת `mg-scr`: `.mg-main` (הסבר+קבוצות, גליל, flex:1) → `#mgResults` (צמוד לתחתית, max-height 44vh) → `.flt-search` בתחתית (אותו פורמט כמו מסך הסינון). נוסף keyboard-lift על visualViewport כמו ב-openFilter.
4. **גרירה תקועה בתוצאות חיפוש המיזוג** (`mgWireDrag`): ‎`.mg-mrow` הוא `touch-action:pan-y` → תנועה אנכית גררה השתלטות-גלילה של הדפדפן → `pointercancel` שאף אחד לא האזין לו → שיבוט תקוע + דליפת מאזינים על document. תוקן: מאזין `pointercancel` שמנקה הכל, ו-`touchmove` לא-פסיבי על השורה שעושה `preventDefault` רק כש-`dragging` (דפוס SortableJS) — הגלילה ברשימה נשארת חופשית כשלא גוררים.

## נעשה ב-Sweep 14 (המשך 2) — ניסוי תזמון המניפה (TODO ‎#3 הישן)
דגל `HBAR_PARALLEL` (מוגדר לפני `closeHbar`, כרגע `true`): פתיחה — `_openDelay=0` ב-`openHbar` (המניפה+X+רקע ביחד עם קיפול הבר); סגירה — `_reopenDelay=0` בענף ה-barBase של `closeHbar` (הבר נפתח ביחד עם קיפול המניפה). `false` מחזיר את התזמון הטורי המקורי. נוספו שני מגני-מירוץ: הניקוי של `botbarCollapse` מדלג אם `barReopening`, והניקוי של `botbarReopen` מדלג אם `barCollapsing` (פתיחה/סגירה מהירות ברצף). ענף ה-nav (ניווט למסך מהמניפה) לא שונה.

**סנכרון משכים (בקשת אלעד אחרי הניסוי):** קבוע `HBAR_T=0.55` (ליד הדגל) — כל ארבע האנימציות מסתיימות בדיוק ב-T: פתיחת מניפה (`--dur=T`, בלי סטגר), סגירת מניפה (delay `(n-1-i)*.04` ב-CSS + `--cdur=T-delay`), קיפול בר (`hmOut`, delay `(N-1-i)*.035` + משך `T-delay`), פתיחת בר (delay `i*.035` + משך `T-delay`). לפני כן: 0.62/0.72/0.40/0.57 — לא מסונכרן. גם `_foldMs`, טיימרי הניקוי, וה-`_openDelay` של מצב הדגל-כבוי נגזרים מ-`HBAR_T`. לכוונון קצב — משנים רק את `HBAR_T`.

**סנכרון ויזואלי (סבב שני):** משכים זהים לא הספיקו — עקומת `hmIn` מגיעה למקום כבר ב~20% מהמשך (השאר ריטוט), בעוד הבר זז לכל אורך המשך. במצב `HBAR_PARALLEL` הפתיחה של המניפה כבר **לא** משתמשת ב-`.run`/`hmIn` אלא בטרנזישן ישיר שמסתיים בסוף המשך. `closeHbar` מנקה את הסגנונות הישירים לפני `closing` כדי ש-`hmOut` יחול. דגל כבוי = חוזר ל-`.run`/`hmIn` המקורי.

**החלטות עיצוב סופיות של אלעד (סבב שלישי):** פתיחה **קפיצית** בשניהם, סגירה **התקפלות** בשניהם: מניפה נפתחת וגם הבר התחתון נפתח עם `cubic-bezier(.34,1.56,.64,1)` (אוברשוט), מניפה מתחילה מ-`scale(.6)`; הסגירה של שניהם נשארת `hmOut` (התכנסות לאייקון). הערת אלעד: העבודה הזו רלוונטית **רק לעמוד הבית**.

**החלפת צד (doHandFlip) — ביטול השהיות (בקשת אלעד):** `HF={CLOSE:550,CLOSE_GAP:0,HOP:550,HOP_GAP:0,BAR_DELAY:800,REOPEN:770}` — CLOSE מיושר למשך קיפול המניפה (HBAR_T), המרווחים בוטלו, REOPEN=220 (סיבוב נקודות)+550 (פתיחת בר). גם דהיית הרקע בענף המיידי של `closeHbar` איבדה את ההשהיה (.28s). הסדר לא שונה.
