# Sweep — המשך עבודה (Handoff, סוף "Sweep 14")

קרא את הקובץ הזה בתחילת כל שיחת Sweep. גרסה נוכחית: **`2026.07.08.0024`** (⚠️ **עדיין לא נפרסה / לא אומתה במכשיר** — צריך `deploy-apps.bat` + "עדכן אפליקציה").

## קבצים ובנייה
- מקור ה-UI: `apps/MyShare.html` (קובץ ענק יחיד). נטען ב-WebView מ-GitHub Pages / מהאסט המובנה `Sweep/android/app/src/main/assets/myshare.html`.
- נייטיב: `apps/Sweep/android/app/src/main/java/com/sweep/ShareActivity.kt` (מחלקת `Bridge` = ממשק JS בשם `AndroidShare`).
- **דיפלוי = דאבל-קליק על `apps/deploy-apps.bat`** (הצג אותו כקרד לחיץ כשמבקשים לפרוס). דוחף את `apps` ל-repo `elad-cmd/elad-apps` → GitHub Actions "Build Sweep APK" בונה APK + מפרסם Release (tag `sweep-latest`) + GitHub Pages.
- **שינויי HTML/CSS** → מגיעים למכשיר דרך "עדכן אפליקציה" (מושך מ-GitHub Pages; עדיין צריך דיפלוי כדי לפרסם). **שינויי Kotlin/נייטיב** → מצריכים בניית APK והתקנה ידנית.
- בכל שינוי: לעדכן `const APP_VERSION=` ב-MyShare.html **וגם** `version.json` יחד.
- חתימת APK = `debug.keystore` מחויב בריפו (יציב → עדכונים לא מוחקים נתונים).
- אפשר לקרוא לוגים של GitHub Actions דרך Claude-in-Chrome (אלעד מחובר) לאבחון כשלי בנייה. תיקון CI שהוחל: `env: ACTIONS_ALLOW_USE_UNSECURE_NODE_VERSION: 'true'` ב-build-sweep.yml.
- **אימות JS: bash רואה עותק מיושן/חתוך של הקובץ ב-Dropbox** (השורות האחרונות נחתכות באמצע!) — אי אפשר להריץ `node --check` אמין על כל הקובץ. לסמוך על כלי ה-host (Read/Grep/Edit) ולאמת כל אזור עריכה מול המקור.
- `fKey`=`statKey`=מספר טלפון גולמי (לא יציב). `healKeys()` (בטעינה) מרפא מפתחות שמורים (members+manual) לפי כל הטלפונים+שם.

## נעשה ב-Sweep 13 (השיחה הזו) — כולל במסך פרטי איש קשר (openContact)
- **אנימציית מסך פרטי → אקורדיון אמיתי:** `scrObjs` פורס עכשיו את ילדי `.ct-card` (במקום כרטיס אחד גדול) לקסקדה; ו-`paintHist` (ההיסטוריה) נדחה ב-`setTimeout(...,80)` כדי שהקריאות הנייטיביות הכבדות (getCallLog/getSmsLog/getNotifLog) לא יחסמו את פריימי הכניסה. פתר את "נכנס/יוצא מאוחר + טקסט-על-טקסט".
- **מולטי-סלקט להיסטוריית מגעים** (`#ctHistory` ב-openContact): לחיצה ארוכה על רשומה → `showSelBar` עם **בחר הכל / הסר** (בלי "העתק"). state מקומי `histSel`/`histSelSet`, מפתח = `evKey(e)` = date|icon|dir. פונקציות: `exitHistSel`, `histRemoveSel`, `updHistBar`.
- **צ'קבוקסים מימין:** בהיסטוריה (flex RTL, ה-`.pk-chk` הוא הילד הראשון). וגם ב**מולטי-סלקט של האחרונים** — הוזז ל-CSS `.row.rec-selrow` (checkbox ל-right:16, `.av` ל-54, `.meta` ל-113). CSS חדש: `.hist-row.sel`.
- **תמונת איש קשר במסך פרטי:** `.ct-av` מקבל עכשיו `position:relative;overflow:hidden` + מציג `person.photo` כמו `avInner`.

## נעשה ב-Sweep 14 — תיקון באג "הרשימה נגללת בלי הטאבים" (pull-to-reach)
הבאג: אחרי חזרה ממסך פרטי למסך הבית, הגלילה-למטה (ה-pull שמוריד את כל המסך) הזיזה רק את הרשימה בלי ה-seg/listbar. סיבת שורש — שלושה כותבי transform מתנגשים: `setPull` מול האנימציות `accIn`/`accOut`. תוקן ב-3 נקודות ב-MyShare.html:
1. **`accIn` (ניקוי ב-1050ms):** המחיקה של transform ל-'' דרסה pull פעיל באמצע מחווה. עכשיו אחרי הניקוי, אם `sc.id==='home'` ו-`pullOffset>0` → `setPull(pullOffset)` מחזיר סנכרון.
2. **`openScreen`:** קודם רק openContact איפס pull; מסכים אחרים (הגדרות/סינון/ארכיון) השאירו `pullOffset>0` בזמן שהאקורדיון דרס את הטאבים → חוסר סנכרון קבוע בחזרה. עכשיו כל פתיחת מסך מ-home מאפסת (`setPull(0)`).
3. **touchmove של ה-pull:** כשמחווה מתחילה, מנקה `transition` גם על seg/listbar (לא רק listEl) כדי שהטאבים יעקבו אחרי האצבע גם בתוך חלון האנימציה של accIn.
